## /n8n:create-workflow

### Steps

1. Ask for workflow purpose and trigger type
2. Search for relevant nodes using n8n MCP
3. Create workflow structure with appropriate pattern
4. Validate configuration
5. Deploy to n8n instance
6. Provide webhook URL if applicable

### Rules
- Always validate before deploying
- Use smart parameters (branch, case) for connections
- Include error handling nodes for production workflows
- Document credential requirements
